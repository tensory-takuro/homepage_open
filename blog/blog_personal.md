特に製造業では、ベテラン社員の**ナレッジ継承**にAIを使う事例が注目されています。

## 論文・学術的根拠の引用ルール

### 基本方針

記事の信頼性を高めるため、**学術論文を積極的に引用**する。特に以下の情報源を活用すること。

### 主要な論文情報源

| 情報源 | URL | 特徴 |
|--------|-----|------|
| **arXiv** | [https://arxiv.org/](https://arxiv.org/) | AI・機械学習の最新論文が集まるオープンアクセスアーカイブ。**最優先で参照** |
| Google Scholar | https://scholar.google.com/ | 幅広い分野の論文検索 |
| Semantic Scholar | https://www.semanticscholar.org/ | AI特化の論文検索 |
| ACL Anthology | https://aclanthology.org/ | 自然言語処理の論文 |

### arXivの活用方法

[arXiv](https://arxiv.org/)は物理学、数学、コンピュータサイエンス、統計学など約240万件の学術論文を無料で公開しているオープンアクセスアーカイブです。AI・機械学習分野の最新研究はほぼここで公開されます。

**検索カテゴリ（AI関連）:**
- `cs.AI` - Artificial Intelligence
- `cs.CL` - Computation and Language（自然言語処理）
- `cs.LG` - Machine Learning
- `cs.IR` - Information Retrieval（RAG関連）
- `cs.MA` - Multiagent Systems（AIエージェント関連）

### 引用の書き方

```markdown
## 良い例（本文中での引用）

RAGの有効性については、Meta社の研究チームによる論文でも実証されています。
この研究では、外部知識を参照することでLLMの回答精度が最大15%向上したと報告されています
（[Lewis et al., 2020](https://arxiv.org/abs/2005.11401)）。

## 良い例（参考文献セクション）

### 参考文献

1. Lewis, P., et al. (2020). "Retrieval-Augmented Generation for Knowledge-Intensive NLP Tasks." *arXiv preprint arXiv:2005.11401*. [https://arxiv.org/abs/2005.11401](https://arxiv.org/abs/2005.11401)

2. Wei, J., et al. (2022). "Chain-of-Thought Prompting Elicits Reasoning in Large Language Models." *arXiv preprint arXiv:2201.11903*. [https://arxiv.org/abs/2201.11903](https://arxiv.org/abs/2201.11903)
```

### 引用時の注意点

| ポイント | 説明 |
|---------|------|
| **分かりやすく言い換える** | 論文の内容をそのまま載せず、読者に伝わる言葉で要約する |
| **数字を活用する** | 「〇〇%向上」「〇〇倍高速」など具体的な数値を引用 |
| **必ずリンクを貼る** | arXivのURLを記載し、興味のある読者が原典を確認できるように |
| **日本語で補足する** | 英語論文の場合は、日本語で内容を解説する |

### 頻出トピックと推奨論文

| トピック | 推奨論文（arXiv） |
|---------|-----------------|
| **RAG** | [Retrieval-Augmented Generation for Knowledge-Intensive NLP Tasks](https://arxiv.org/abs/2005.11401) |
| **Chain-of-Thought** | [Chain-of-Thought Prompting Elicits Reasoning in Large Language Models](https://arxiv.org/abs/2201.11903) |
| **AIエージェント** | [A Survey on Large Language Model based Autonomous Agents](https://arxiv.org/abs/2308.11432) |
| **プロンプトエンジニアリング** | [A Survey on In-context Learning](https://arxiv.org/abs/2301.00234) |
| **LLM基礎** | [Attention Is All You Need](https://arxiv.org/abs/1706.03762)（Transformer原論文） |
| **GPT系** | [Language Models are Few-Shot Learners](https://arxiv.org/abs/2005.14165)（GPT-3論文） |

### 引用を入れるタイミング

記事内で以下の場面では、**必ず論文を引用**すること：

1. **技術の効果を主張するとき**
   - 例：「RAGを導入すると回答精度が向上する」→ 論文データで裏付け

2. **新しい概念を説明するとき**
   - 例：「AIエージェントとは」→ 定義の出典を明記

3. **トレンドを語るとき**
   - 例：「2024年はAIエージェント元年」→ 調査論文やレポートを引用

4. **比較・評価をするとき**
   - 例：「AよりBの方が精度が高い」→ ベンチマーク論文を引用

### 論文が見つからない場合の代替ソース

| 情報源 | 用途 |
|--------|------|
| 企業テックブログ | Google AI Blog, OpenAI Blog, Meta AI Blog |
| 公式ドキュメント | OpenAI Docs, Anthropic Docs |
| 業界レポート | Gartner, McKinsey, 野村総研 |
| ニュースサイト | TechCrunch, The Verge, ITmedia |
