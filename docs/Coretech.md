# Tensory Core Technology - ナレッジエンジニアリング

## 概要

Tensoryのコアテクノロジーである**ナレッジエンジニアリング**は、企業に眠る暗黙知を形式知へと変換し、AIと人間が共に成長する「知の循環」を実現する技術体系です。

---

## 新しい知の循環 - 現代のSECIモデル

### 従来のSECIモデル
野中郁次郎が提唱した知識創造プロセス：
- **S**ocialization（共同化）: 暗黙知から暗黙知へ — 共有体験を通じて、暗黙知を他者に伝達する
- **E**xternalization（表出化）: 暗黙知から形式知へ — 暗黙知を言語化・概念化して形式知に変換する
- **C**ombination（連結化）: 形式知から形式知へ — 異なる形式知を組み合わせて新たな形式知を創造する
- **I**nternalization（内面化）: 形式知から暗黙知へ — 形式知を実践を通じて暗黙知として体得する

### 循環のポイント

1. **暗黙知の形式知化（表出化の加速）**
   - ベテランの経験・勘をAIが理解できる形で記録
   - 自然言語でのインタラクションにより、暗黙知の抽出が容易に

2. **AIによる知識の増幅（連結化の強化）**
   - 構造化された知識をAIが分析・統合
   - 人間には見えなかったパターンや関連性を発見

3. **パーソナライズされた知識提供（内面化の促進）**
   - 個人の状況に応じた知識の提示
   - 必要なときに必要な情報を適切な形で提供

4. **継続的な学習と改善**
   - AIの回答に対するフィードバックが新たな知識として蓄積
   - 利用されるほど精度が向上する自己強化ループ


## 2つの柱

### 1. データストラクチャリング（Data Structuring）

#### 定義
未加工の情報（ドキュメント、会話ログ、業務マニュアル、ベテランのノウハウなど）を整形・構造化し、AIが効果的に利活用できる「ナレッジ」へと加工するプロセス。

#### なぜ重要か
- 企業の知識は多くの場合、散在・非構造化されている
- 生のデータのままではAIが正確に理解・活用できない
- 適切に構造化することで、検索精度・回答精度が飛躍的に向上

#### プロセス
1. **収集（Collection）**
   - 社内ドキュメント、Wiki、チャット履歴、録音データなど多様なソースから情報を収集
   - 暗黙知を持つ人材へのインタビュー・観察

2. **整形（Normalization）**
   - フォーマットの統一
   - ノイズの除去
   - 重複情報の整理

3. **構造化（Structuring）**
   - メタデータの付与（カテゴリ、タグ、関連性）
   - 階層構造・関係性の定義
   - セマンティックな意味付け

4. **最適化（Optimization）**
   - AIモデルに適したチャンク分割
   - エンベディング生成のための前処理
   - 検索性能を最大化するインデックス設計

#### 成果物
- 構造化されたナレッジベース
- AIが理解可能な形式のドキュメント群
- 検索・参照に最適化されたデータセット

---

### 2. コンテキストエンジニアリング（Context Engineering）

#### 定義
構造化されたナレッジを最適に利活用するため、「どのように知識を蓄積し、AIエージェントに活用させるか」を設計・実装するプロセス。

#### なぜ重要か
- 優れたデータがあっても、適切なコンテキストなしにはAIは最適な回答を生成できない
- 同じ質問でも、状況・文脈によって必要な情報は異なる
- AIエージェントが「賢く」振る舞うには、適切なコンテキスト設計が不可欠

#### 主要コンポーネント

1. **コンテキストウィンドウ設計**
   - AIに渡す情報の選択と優先順位付け
   - 動的なコンテキスト生成

2. **プロンプトアーキテクチャ**
   - システムプロンプトの設計
   - ユーザークエリの解釈と拡張
   - 出力フォーマットの制御

3. **メモリマネジメント**
   - 短期記憶（会話履歴）
   - 長期記憶（ナレッジベース）
   - 作業記憶（タスク固有の情報）

4. **エージェントオーケストレーション**
   - 複数AIエージェントの協調
   - タスク分解と委譲
   - 結果の統合と検証

#### 実装パターン
- **RAG（Retrieval-Augmented Generation）**: ナレッジベースから関連情報を検索し、生成に活用
- **Agent Memory**: 対話履歴や学習内容を蓄積し、パーソナライズされた応答を生成
- **Multi-Agent System**: 専門性の異なる複数エージェントが協力して複雑なタスクを解決

---


Tensoryのナレッジエンジニアリングは、単なるAI導入ではなく、組織の知識を持続的に活用・発展させるための基盤技術です。

**データストラクチャリング**で知識を整え、**コンテキストエンジニアリング**で最適に活用し、**新しい知の循環**で人とAIが共に成長する。

これがTensoryが提供するコアバリューです。
